# Distillation Trainer

[![All_models-Distillation-blue](https://img.shields.io/badge/All_models-Distillation-blue)](https://huggingface.co/models?other=distillation,trl)

## Overview

The Distillation Trainer implements on-policy knowledge distillation, introduced in [On-Policy Distillation of Language Models: Learning from Self-Generated Mistakes](https://huggingface.co/papers/2306.13649) by Rishabh Agarwal, Nino Vieillard, Yongchao Zhou, Piotr Stanczyk, Sabela Ramos, Matthieu Geist, and Olivier Bachem.

> Knowledge distillation (KD) is widely used for compressing a teacher model to reduce its inference cost and memory footprint, by training a smaller student model. However, current KD methods for auto-regressive sequence models suffer from distribution mismatch between output sequences seen during training and those generated by the student during inference. To address this issue, we introduce Generalized Knowledge Distillation (GKD). Instead of solely relying on a fixed set of output sequences, GKD trains the student on its self-generated output sequences by leveraging feedback from the teacher on such sequences. Unlike supervised KD approaches, GKD also offers the flexibility to employ alternative loss functions between the student and teacher, which can be useful when the student lacks the expressivity to mimic the teacher's distribution.

The idea is a loop of three steps, repeated every training step:

1. The **student generates** completions from its own policy.
2. The **teacher scores** them, i.e. it runs a forward pass over the student's own tokens. The teacher never generates.
3. The student is trained to **minimize a per-token divergence** against the teacher's next-token distribution, at the positions the student actually visited.

Training on the student's own samples is what separates this from supervised fine-tuning on teacher-generated text: the student is corrected on the states it actually reaches at inference time, rather than on trajectories it may never produce.

## Quick start

```python
from datasets import load_dataset
from trl import DistillationConfig, DistillationTrainer

dataset = load_dataset("trl-lib/DeepMath-103K", split="train")

trainer = DistillationTrainer(
    model="Qwen/Qwen2.5-0.5B-Instruct",
    teacher_model="Qwen/Qwen2.5-7B-Instruct",
    args=DistillationConfig(output_dir="Qwen2.5-0.5B-Distill"),
    train_dataset=dataset,
)
trainer.train()
```

## Usage tips

### Choosing the divergence

`beta` selects which divergence is minimized. Note that, unlike in [`GRPOConfig`], `beta` is *not* a KL-penalty coefficient here:

| `beta` | Divergence | Behavior |
| --- | --- | --- |
| `0.0` | Forward KL, `KL(teacher ‖ student)` | Mass-covering: the student is penalized for missing any mode the teacher has. Used by Qwen3. |
| `1.0` (default) | Reverse KL, `KL(student ‖ teacher)` | Mode-seeking: the student concentrates on the teacher's main modes. Used by DeepSeek-V4 and GLM-5. |
| in between | Generalized JSD | Interpolates between the two. The GKD paper's default is `0.5`. |

Reverse KL is the default because it is what recent large models use, and because it pairs naturally with on-policy sampling: you only need the teacher's probabilities at the student's own samples.

### Shared vocabulary

The divergence compares the *full* next-token distributions of the student and the teacher, so both must be defined over the same vocabulary. Passing a teacher with a different `vocab_size` raises an error. For cross-tokenizer distillation, use [`experimental.gold.GOLDTrainer`] instead.

### Speeding up generation

The student generating completions is the dominant cost of a training step. Set `use_vllm=True` to generate with vLLM instead of `transformers`:

```python
from trl import DistillationConfig

training_args = DistillationConfig(output_dir="Qwen2.5-0.5B-Distill", use_vllm=True)
```

See [Speeding up training](speeding_up_training) for details.

### Memory

The divergence needs the full `(batch, seq_len, vocab_size)` distribution for *both* models, which is far too large to materialize at a realistic vocabulary size. The trainer never does: the vocabulary projection is computed in chunks of tokens, each chunk kept alive only for its own forward/backward via gradient checkpointing, so peak memory scales with the chunk size rather than the sequence length.

### Expected dataset type

The dataset must contain a `"prompt"` column; the student generates the completions itself, so no completion column is needed. Both [standard](dataset_formats#standard) and [conversational](dataset_formats#conversational) [prompt-only](dataset_formats#prompt-only) formats are supported.

```python
{"prompt": "The sky is"}
```

```python
{"prompt": [{"role": "user", "content": "What color is the sky?"}]}
```

## Example script

Use [`examples/scripts/distillation.py`](https://github.com/huggingface/trl/blob/main/examples/scripts/distillation.py) to launch distillation training from the command line:

```bash
python examples/scripts/distillation.py \
    --model_name_or_path Qwen/Qwen2.5-0.5B-Instruct \
    --teacher_model_name_or_path Qwen/Qwen2.5-1.5B-Instruct \
    --dataset_name trl-lib/DeepMath-103K \
    --learning_rate 1e-5 \
    --per_device_train_batch_size 4 \
    --output_dir Qwen2.5-0.5B-Distill
```

## Logged metrics

* `num_tokens`: Total number of tokens processed so far, prompts plus completions.
* `entropy`: Average entropy of the student's next-token distribution over the completion tokens.
* `completions/mean_length`: Average length of the generated completions.
* `completions/min_length`: Minimum length of the generated completions.
* `completions/max_length`: Maximum length of the generated completions.
* `completions/clipped_ratio`: Fraction of completions that hit `max_completion_length` without emitting EOS.
* `completions/mean_terminated_length`: Average length of the completions that did emit EOS.
* `completions/min_terminated_length`: Minimum length of the completions that did emit EOS.
* `completions/max_terminated_length`: Maximum length of the completions that did emit EOS.
